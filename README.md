# Kids Tesla Art

**Turn your child's drawing into a custom Tesla wrap.**

Kids Tesla Art is an open-source web application that transforms a child's hand-colored Tesla template into a custom wrap file for the Tesla Toybox Paint Shop. No design skills or Photoshop required.

---

## How It Works

1. **Download & Print** — Get the coloring-book-style Tesla template (PDF) from the site
2. **Color** — Let your child color it with crayons, markers, or watercolors
3. **Upload** — Take a photo and upload it on your phone or desktop
4. **Download** — Receive a Tesla-ready PNG file (1024×1024, ≤1MB)
5. **Apply** — Transfer to your Tesla via USB drive through Toybox → Paint Shop → Wraps

## Local Development

### Prerequisites
- Docker & Docker Compose
- Node.js 20+
- Python 3.11+

### Quick Start with Docker Compose
```bash
git clone https://github.com/sung-park/kids-tesla-art.git
cd kids-tesla-art

# Generate template files first
cd scripts
pip install opencv-contrib-python Pillow reportlab
python generate_templates.py
cd ..

# Start all services
docker compose -f docker-compose.dev.yml up --build
```

Open [http://localhost](http://localhost) in your browser.

### Running Services Individually

**Backend:**
```bash
cd backend
pip install -r requirements-dev.txt
uvicorn app.main:app --reload --port 8000
```

**Frontend:**
```bash
cd frontend
npm install
npm run dev
```

### Running Tests

**Backend:**
```bash
cd backend
pytest -v
```

**Frontend:**
```bash
cd frontend
npm test
```

## Deployment (OCI ARM Free Tier)

### One-time Server Setup

1. Provision `VM.Standard.A1.Flex` on Oracle Cloud (4 OCPU, 24GB RAM, Ubuntu 22.04)

2. Open firewall ports in OCI Security Lists: TCP 22, 80, 443

3. Open ports in OS iptables:
   ```bash
   sudo iptables -I INPUT -p tcp --dport 80 -j ACCEPT
   sudo iptables -I INPUT -p tcp --dport 443 -j ACCEPT
   sudo apt install iptables-persistent -y
   ```

4. Install Docker:
   ```bash
   curl -fsSL https://get.docker.com | sh
   sudo usermod -aG docker ubuntu
   ```

5. Clone repo and set up environment:
   ```bash
   git clone https://github.com/sung-park/kids-tesla-art.git ~/kids-tesla-art
   cd ~/kids-tesla-art
   echo "DOCKERHUB_USERNAME=your-username" > .env
   ```

6. Generate templates (first time only):
   ```bash
   pip3 install opencv-contrib-python Pillow reportlab
   python3 scripts/generate_templates.py
   ```
   Then replace `backend/app/templates/*.png` with official Tesla UV templates from
   [teslamotors/custom-wraps](https://github.com/teslamotors/custom-wraps).

### Configure GitHub Secrets

In your GitHub repository settings, add:

| Secret | Value |
|--------|-------|
| `DOCKERHUB_USERNAME` | Your DockerHub username |
| `DOCKERHUB_TOKEN` | DockerHub access token |
| `OCI_HOST` | Your OCI server public IP |
| `OCI_SSH_KEY` | Private SSH key for the `ubuntu` user |

### Deploy

Push to `main` to trigger automatic deployment:
```bash
git push origin main
```

The GitHub Actions workflow will:
1. Run all tests
2. Build multi-platform Docker images (amd64 + arm64)
3. Push to DockerHub
4. SSH into OCI and run `docker compose pull && docker compose up -d`

## Template Attribution

The UV template PNG files in `backend/app/templates/` should be replaced with official
templates from [teslamotors/custom-wraps](https://github.com/teslamotors/custom-wraps).
The placeholder files generated by `scripts/generate_templates.py` are for development only.
